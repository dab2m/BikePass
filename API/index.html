JSON syntax and responses for API

For Android App GET and POST request should be targeted to app.php file
For Bikes GET and POST request should be targeted to bike.php file
You can find JSON formats on left and their response formats on right.Under response format status codes and example messages are listed
You can copy example JSON formats with copy button

Register a new User

{

"username": "mustafa",

"password": "sEcreTp@ssW0rd",

"email": "mustafa@bestcompany.com"

}

To register a new user three parameters are required. username, email and password.
Username and email should be exclusive to each user
Username should be between 2-20 lowercase letters
Password should be between 8-12 characters with upper and lowercase letters and numbers.Password can alson include !@#%$ characters

Response for Register

{

"status": "0",

"message": "Account is created! Welcome to BikePass mustafa"

}

Status Codes

  • "0" "Account is created!"
  • "1" "Username is not in all lowercase or not between 2-20"
  • "2" "Email is invalid"
  • "3" "Password doesn't meet requirements"
  • "4" "Email exists"
  • "5" "Username exists"

Login

{

username: mustafa,

password: sEcreTp@ssW0rd,

}

To login two parameters are required. username and password.

Response for Login

{

"status": "0",

"message": "Login succesful. Welcome back mustafa"

}

Status Codes

  • "0" "Login successful!"
  • "1" "Wrong username,password combination"
  • "2" "No user found for username mustafa"

Send Location

{

lat: 39.9210241,

long: 32.7974713

}

To receive array of available bikes, two parameters are required. latitude and longtitude.
bikes is a json object not a json array!

Response for Send Location

{

"status": "0",

"message": "Returned object of bikes",

"bikes":{"0":{"name": "bike1", "lat":40.1231234,"long": 33.8211234 } ,
"1":{"name": "bike2", "lat":41.1231234,"long": 34.8211234 }}

}

Status Codes

  • "0" "Returned object of bikes"
  • "1" "No available bike near user"

Send Bike Data

{

username: mustafa,

bike_id: 1,

bike_time: 1200,

bike_km: 5

}

To add data of a bike usage, four parameters are required. username bike_id bike_time and bike_km
bike_time should be in seconds
bike_km should be in kilometers

Response for Send Bike Data

{

"status": "0",

"message": "Data stored and bike1 is available again"

}

Status Codes

  • "0" "Data stored and bike and bike1 is available again"
  • "1" "Can't find user with username mustafa"
  • "2" "Invalid bike id!"
  • "3" "Non-busy bike detected!"
  • "4" "Can't add bike using data!"
  • "5" "Can't update bikes status!"
