JSON syntax and responses for API

For Android App GET and POST request should be targeted to app.php file
For Bikes GET and POST request should be targeted to bike.php file
You can find JSON formats on left and their response formats on right.Under response format status codes and example messages are listed
You can copy example JSON formats with copy button

Register a new User

{

"username": "mustafa",

"password": "sEcreTp@ssW0rd",

"email": "mustafa@bestcompany.com"

}

To register a new user three parameters are required. username, email and password.
Username and email should be exclusive to each user
Username should be between 2-20 lowercase letters
Password should be between 8-12 characters with upper and lowercase letters and numbers.Password can alson include !@#%$ characters

Response for Register

{

"status": "0",

"message": "Account is created! Welcome to BikePass mustafa"

}

Status Codes

  • "0" "Account is created!"
  • "1" "Username is not in all lowercase or not between 2-20"
  • "2" "Email is invalid"
  • "3" "Password doesn't meet requirements"
  • "4" "Email exists"
  • "5" "Username exists"

Send Location

{

"lat": 39.9210241,

"long": 32.7974713,

"usernamebikes": "mustafa

}

To receive array of available bikes, three parameters are required. latitude longtitude and usernamebikes.
Checks if usernamebikes has a request if so returns latitude longtitude values of it

Response for Send Location

{

"status": "0",

"message": "Returned object of bikes",

"bikes":[
{"name":"bike51","lat":"39.9249","long":"32.8047","status":"1"},{"name":"bike61","lat":"39.9173","long":"32.8009","status":"1"}],
"lat":"39.921",
"long":"32.7975"

}

Status Codes

  • "0" "Returned object of bikes"
  • "1" "No user found with username mustafa"
  • "2" "Returned object of bikes with no request"

Send Bike Data

{

"username": mustafa,

"bike_id": 101,

"lat": 39.9249,

"long": 32.8047,

"bike_time": 1200

}

To add data of a bike usage, four parameters are required. username bike_id bike_time and bike_km
bike_time should be in seconds. If bike is in area of a hotpoint return 1000 credits. If bike is in area of a request return credit scaled with current time and request time difference.

Response for Send Bike Data

{

"status": "0",

"message": "Data stored and bike1 is available again",

"credit": "1000"

}

Status Codes

  • "0" "Data stored and bike and bike1 is available again"
  • "1" "Can't find user with username mustafa"
  • "2" "Invalid bike id!"
  • "3" "Non-busy bike detected!"
  • "4" "Can't add bike using data!"
  • "5" "Can't update bikes status!"

Unlock bike

{

"bike_id": 101,

"lat": 39.9249,

"long": 32.8047,

"username": "mustafa"

}

After scanning qr code, to update bike's status on database, provide two variables: bike_id retrieved from qr code and username to update current user of this bike. Also updates hotpoints' frequencies. If given user has a request, also deletes that request

Response for Unlock

{

"status": "0",

"message": "Bike is unlocked"

}

Status Codes

  • "0" "Bike is unlocked"
  • "1" "Unidentified bike_id"
  • "2" "Unknown username mustafa"
  • "3" "Bike is not available"
  • "4" "Database error! Couldn't update bike's status"

Get Weekly Data

{

"username": "mustafa",

"type": "distance"

}

From this week's Monday to current day, return total usage time or kilometers. For type use time for seconds or distance for kilometers. Use paramater "all": true to list all data without weekly limit User paramater "date": true to list date as April 02 2020 instead of Thursday

Response for Weekly data

{

"status": "0",

"message": "Returned data",

"data": [
  {"day":"Tuesday","bike_km":"5"},
  {"day":"Wednesday","bike_km":"5"}
]

}

Status Codes

  • "0" "Returned data"
  • "1" "Unknown username mustafa"
  • "2" "No data between 2020-03-23 - 2020-03-25"
